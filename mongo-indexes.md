<h2><a id="user-content-創建index" class="anchor" aria-hidden="true" href="#創建index"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>創建index</h2>
<p>要創建index的指令時：</p>
<pre><code>db.collection.createIndex( &lt;key and index type specification&gt;, &lt;options&gt; )

</code></pre>
<p>範例:建個單一降序key的index在name領域</p>
<pre><code>db.collection.createIndex( { name: -1 } )
</code></pre>
<h2><a id="user-content-index-種類" class="anchor" aria-hidden="true" href="#index-種類"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>index 種類</h2>
<p>mongoDB 提供各種不同的index種類來處理不同的query跟data。以下single field, compound index, 會描述的比較細，其他的會做一個比較膚淺的描述。</p>
<h3><a id="user-content-single-field" class="anchor" aria-hidden="true" href="#single-field"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Single Field</h3>
<p>這種索引是使用者可以建的，一次可以索引一個領域。</p>
<ul>
<li>假如有個collection 命名為 records 抱有以下的資料：</li>
</ul>
<pre><code>{
 "_id": ObjectId("570c04a4ad233577f97dc459"),
 "score": 1034,
 "location": { state: "NY", city: "New York" }
}
</code></pre>
<p>以下的指令建個升序的index：</p>
<pre><code>db.records.createIndex( { score: 1 } )
</code></pre>
<p>數字是代表那是給那個field的哪種index。例如，1 是代表升序排序回傳的資料，-1是代表降序。
被創建的index會提供在score領域作select的動作的query，像是：</p>
<pre><code>db.records.find( { score: 2 } )
db.records.find( { score: { $gt: 10 } } )
</code></pre>
<h4><a id="user-content-在嵌入式field建index" class="anchor" aria-hidden="true" href="#在嵌入式field建index"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>在嵌入式field建index</h4>
<p>你可以在嵌入式的資料領域建index，也可以指頂級的領域。</p>
<ul>
<li>假如有個collection 命名為 records 抱有以下的資料：</li>
</ul>
<pre><code>{
  "_id": ObjectId("570c04a4ad233577f97dc459"),
  "score": 1034,
  "location": { state: "NY", city: "New York" }
}
</code></pre>
<p>以下的指令建個index在location.state領域。</p>
<pre><code>db.records.createIndex( { "location.state": 1 } )
</code></pre>
<p>被創建的index會提供在location.state領域作select的query，像是：</p>
<pre><code>db.records.find( { "location.state": "CA" } )
db.records.find( { "location.city": "Albany", "location.state": "NY" } )
</code></pre>
<h4><a id="user-content-在嵌入式資料建index" class="anchor" aria-hidden="true" href="#在嵌入式資料建index"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>在嵌入式資料建index</h4>
<p>你也可以在整個嵌入式的資料建index。</p>
<ul>
<li>假如有個collection 命名為 records 抱有以下的資料：</li>
</ul>
<pre><code>{
  "_id": ObjectId("570c04a4ad233577f97dc459"),
  "score": 1034,
  "location": { state: "NY", city: "New York" }
}
</code></pre>
<p>location是個嵌入式的資料，裡面有綁定city跟state。以下的指令建個index在location的整個field。</p>
<pre><code>db.records.createIndex( { location: 1 } )
</code></pre>
<p>被創建的index會提供在location領域作select的動作的query，像是：</p>
<pre><code>db.records.find( { location: { city: "New York", state: "NY" } } )
</code></pre>
<h3><a id="user-content-compound-field" class="anchor" aria-hidden="true" href="#compound-field"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Compound field</h3>
<p>如果single field 不達到使用者的需求，compound index 可以指多數的領域。例如，如果有個compound index是 { userid: 1, score: -1 }，指標就會重userid排序然後在每個userid的值內，排序score。</p>
<h4><a id="user-content-建compound-index" class="anchor" aria-hidden="true" href="#建compound-index"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>建compound index</h4>
<p>以下是compound index的指令原型：</p>
<pre><code>db.collection.createIndex( { &lt;field1&gt;: &lt;type&gt;, &lt;field2&gt;: &lt;type2&gt;, ... } )
</code></pre>
<p>以下有個collection命名為products包以下的資料：</p>
<pre><code>{
"_id": ObjectId(...),
"item": "Banana",
"category": ["food", "produce", "grocery"],
"location": "4th Street Store",
"stock": 4,
"type": "cases"
}
</code></pre>
<p>以下的指令會建個升序的index在item跟stock領域：</p>
<pre><code>db.products.createIndex( { "item": 1, "stock": 1 } )
</code></pre>
<p>領域list的順序很重要。index會先排序item領域的value，然後在item field的每個直，排序stock field的直。被創建的index會提供在item領域或item跟stock一起領域的query，像是：</p>
<pre><code>db.products.find( { item: "Banana" } )
db.products.find( { item: "Banana", stock: { $gt: 5 } } )
</code></pre>
<h4><a id="user-content-排序順序" class="anchor" aria-hidden="true" href="#排序順序"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>排序順序</h4>
<p>indexes以升序（1）或降序（-1）排序。對single-field index， key的排序的順序不會影響因為MONGODB可以橫過index的每一邊。不過在compound index排序順序會影響到他是否可以提供sort的動作。
以下有個collection 命名為 events， 有個document包username 跟 date 的領域。應用程式可以執行回傳username得值先升序排序然後在date的值降序的query，像：</p>
<pre><code>db.events.find().sort( { username: 1, date: -1 } )
</code></pre>
<p>或是username 先降序排序在升序date。</p>
<pre><code>db.events.find().sort( { username: -1, date: 1 } )
</code></pre>
<p>以下的index都可以提供這兩種排序的指令：</p>
<pre><code>db.events.createIndex( { "username" : 1, "date" : -1 } )
</code></pre>
<p>但絕對不能提供這種username跟date都升序的：</p>
<pre><code>db.events.find().sort( { username: 1, date: 1 } )
</code></pre>
<h4><a id="user-content-prefixes" class="anchor" aria-hidden="true" href="#prefixes"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Prefixes</h4>
<p>index 的prefixes 是被指標的field subset的「開頭」。例如以下的compound index：</p>
<pre><code>{ "item": 1, "location": 1, "stock": 1 }
</code></pre>
<p>這個index的prefixes就是：</p>
<ul>
<li>{ item: 1 }</li>
<li>{ item: 1, location: 1 }</li>
</ul>
<p>MONGODB 可利用index在index prefixes作query，以下的field可以利用index做query的動作。</p>
<ul>
<li>item field,</li>
<li>item field 跟 location field,</li>
<li>item field 跟 location field 跟 the stock field.</li>
</ul>
<p>MONGODB 也可以利用index在 item 跟 stock 來作query，因為item定義成個prefix。</p>
<p>但mongoDB不能提供query在以下的field，因為沒有item 的field，其他的都沒被定義成prefix：</p>
<ul>
<li>location field,</li>
<li>stock field, 或</li>
<li>location and stock fields.</li>
</ul>
<h3><a id="user-content-multikey-index" class="anchor" aria-hidden="true" href="#multikey-index"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Multikey Index</h3>
<p>是用來指標有array的field。如果你索引一個array 的 field，mongoDB會建分開的index給每個array的元素。</p>
<h3><a id="user-content-geospatial-index" class="anchor" aria-hidden="true" href="#geospatial-index"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Geospatial Index</h3>
<p>為了達成有效率的地理空間作標數據的query，MONGODB提供兩種特別的index： 2d indexes會利用平面幾何來呈現結果， 2dsphere indexes 利用球面幾何來呈現結果。</p>
<h3><a id="user-content-text-indexes" class="anchor" aria-hidden="true" href="#text-indexes"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Text Indexes</h3>
<p>mongoDB提供文件索引來尋找collection裡面的參數。</p>
<h3><a id="user-content-hashed-index" class="anchor" aria-hidden="true" href="#hashed-index"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Hashed Index</h3>
<ul>
<li>雜湊索引會把輸入被索引的field 作雜湊的動作。</li>
<li>雜湊索引提供sharding的方法（跨多台機器分發數據的方法）hashed shard 的 key</li>
</ul>
<h4><a id="user-content-hashing-的-function" class="anchor" aria-hidden="true" href="#hashing-的-function"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Hashing 的 Function</h4>
<p>雜湊索引利用hashing 的 function 來計算雜湊的index field 的直 。 Hashing 沒提供 multikey的index（字串）。</p>
<h4><a id="user-content-建hashed-key" class="anchor" aria-hidden="true" href="#建hashed-key"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>建hashed key</h4>
<p>以下的範例：</p>
<pre><code>db.collection.createIndex( { _id: "hashed" } )
</code></pre>
<h2><a id="user-content-index-的屬性" class="anchor" aria-hidden="true" href="#index-的屬性"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>index 的屬性</h2>
<h3><a id="user-content-ttlindex" class="anchor" aria-hidden="true" href="#ttlindex"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>TTL　INDEX</h3>
<p>TTL 索引會照著使用者設定的時間之後自動刪除文件。數據期限對某些資訊種類是很有用的，例如機器產生的event data，logs 還有資訊的session，他們存在database都有限制時間。</p>
<p>要建個TTL 索引可下db.collection.createIndex()方法在加上expireAfterSeconds的選定在領域，裡面的直必須是個DATE 或者是date value 的陣列。</p>
<p>例如：以下會在eventLog的collection建個TTL 索引在lastModifiedDate的field的指令：</p>
<pre><code>db.eventlog.createIndex( { "lastModifiedDate": 1 }, { expireAfterSeconds: 3600 } )
</code></pre>
<h4><a id="user-content-限制" class="anchor" aria-hidden="true" href="#限制"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>限制</h4>
<ul>
<li>TTL 索引是single field的索引，compound 索引不提供TTL的功能，他會忽視expireAfterSeconds的選定。</li>
<li>_id field不提供TTL索引</li>
<li>你不能在個 <a href="https://docs.mongodb.com/manual/core/capped-collections/" rel="nofollow">capped collection</a> 建TTL索引因為mongoDB不能在 capped collection 刪除文件。</li>
<li>你不能用createIndex()來改變已經存在的索引的expireAfterSeconds。請利用 collMod database的指令，或者是刪除目前的索引然後重新建。</li>
<li>如果一個不是TTL的索引已存在一個field，你不能在這個field建TTL。要改變這個部分請先刪除目前的index然後在重建個索引加上expireAfterSeconds的選定。</li>
</ul>
<h3><a id="user-content-unique-indexes" class="anchor" aria-hidden="true" href="#unique-indexes"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Unique indexes</h3>
<p>Unique index會確保被索引的field不會有一樣的直。</p>
<h4><a id="user-content-建unique-index" class="anchor" aria-hidden="true" href="#建unique-index"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>建unique index</h4>
<p>下db.collection.createIndex()在加上unique的選定：</p>
<pre><code>db.collection.createIndex( &lt;key and index type specification&gt;, { unique: true } )

</code></pre>
<h5><a id="user-content-single-field-的unique-index" class="anchor" aria-hidden="true" href="#single-field-的unique-index"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>single field 的unique index</h5>
<p>以下我們在members的collection裡建個unique索引在user_id：</p>
<pre><code>db.members.createIndex( { "user_id": 1 }, { unique: true } )
</code></pre>
<h5><a id="user-content-compound-field-的-unique-index" class="anchor" aria-hidden="true" href="#compound-field-的-unique-index"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>compound field 的 unique index</h5>
<p>以下我們在groupNumber, lastname, firstname 建個unique索引：</p>
<p>db.members.createIndex( { groupNumber: 1, lastname: 1, firstname: 1 }, { unique: true } )</p>
<p>再舉一個例子：</p>
<pre><code>{ _id: 1, a: [ { loc: "A", qty: 5 }, { qty: 10 } ] }
</code></pre>
<p>建個unique compound的multikey索引在a.loc 跟 a.qty:</p>
<pre><code>db.collection.createIndex( { "a.loc": 1, "a.qty": 1 }, { unique: true } )
</code></pre>
<p>創建的unique索引能允許以下資料的插入到collection因為這個索引強制 a.loc 跟 a.qty 的組合獨特性。</p>
<pre><code>db.collection.insert( { _id: 2, a: [ { loc: "A" }, { qty: 5 } ] } )
db.collection.insert( { _id: 3, a: [ { loc: "A", qty: 10 } ] } )
</code></pre>
<h3><a id="user-content-partial-indexes" class="anchor" aria-hidden="true" href="#partial-indexes"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Partial Indexes</h3>
<p>它只會索引符合過濾expression的文件。partial 索引會有比較低的存儲要求跟少的績效成本來作索引創建跟維護的動作因為它會索引文件的子集。</p>
<h4><a id="user-content-建partial-index" class="anchor" aria-hidden="true" href="#建partial-index"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>建partial index</h4>
<p>下db.collection.createIndex()的方法加上partialFilterExpression的選定。partialFilterExpression 可接受以下的條件：</p>
<ul>
<li>equality 的expressions(例如：field： value 或是用 $eq 運算符）</li>
<li>$exists: true 的expression,</li>
<li>$gt, $gte, $lt, $lte 的 expressions</li>
<li>$type expressions,</li>
<li>$and 運算符用在頂層</li>
</ul>
<p>例如，以下的的指令是來索引文件裡面的大於5的rating:</p>
<pre><code>db.restaurants.createIndex(
   { cuisine: 1, name: 1 },
   { partialFilterExpression: { rating: { $gt: 5 } } }
)
</code></pre>
<h3><a id="user-content-sparse-index" class="anchor" aria-hidden="true" href="#sparse-index"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Sparse index</h3>
<p>跟Partial index很像但sparse索引只會在有FIELD的document建索引key。</p>
<h2><a id="user-content-在大型collection操作index" class="anchor" aria-hidden="true" href="#在大型collection操作index"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>在大型collection操作index</h2>
<p>預設中，如果在個很多資料的collection建索引其他的預算就會被鎖住。在索引沒被建完之前資料庫作讀跟寫的動作。</p>
<h4><a id="user-content-後台建設" class="anchor" aria-hidden="true" href="#後台建設"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>後台建設</h4>
<p>如果有可能長時間運行的索引創建操作，請考慮後台創建的選定，這樣在index創建的進行中mongoDB才仍然可用。</p>
<p>以下的範例在zipcode的後台建個索引。</p>
<pre><code>db.people.createIndex( { zipcode: 1 }, { background: true } )
</code></pre>
<p>你也可以和其他的參數結合：</p>
<pre><code>db.people.createIndex( { zipcode: 1 }, { background: true, sparse: true } )
</code></pre>

<h3><a id="user-content-查詢存在的index" class="anchor" aria-hidden="true" href="#查詢存在的index"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>查詢存在的index</h3>
<p>要顯示collection裡所有的index，請下db.collection.getIndexes() ，例如以下的指令會回傳people collection 所有的索引：</p>
<pre><code>db.people.getIndexes()
</code></pre>
<p>顯示資料庫所有的index的指令：</p>
<pre><code>db.getCollectionNames().forEach(function(collection) {
   indexes = db[collection].getIndexes();
   print("Indexes for " + collection + ":");
   printjson(indexes);
});
</code></pre>
<h3><a id="user-content-刪除index" class="anchor" aria-hidden="true" href="#刪除index"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>刪除index</h3>

<pre><code>db.accounts.dropIndex( { "tax-id": 1 } )
